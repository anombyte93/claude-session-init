The Problem

No Persistent Memory

Every session starts from zero. Context, decisions, and progress are lost. You re-explain the same things every time.

Project Sprawl

Files accumulate at the project root with no organization. Scripts, docs, configs, and logs pile up across sessions.

No "Done" Signal

Projects have goals, but no mechanism to track progress or verify completion. Are you actually done, or just stuck?

How It Works

One command to rule your project lifecycle.

1

Run /start in a new project

Atlas detects you're starting fresh and enters Init Mode.

→ Captures your soul purpose
→ Bootstraps 5-file memory bank
→ Organizes scattered files
→ Generates CLAUDE.md
2

Work across multiple sessions

Each time you return, run /start to enter Reconcile Mode.

→ Validates and repairs session files
→ Refreshes CLAUDE.md from codebase
→ Self-assesses: incomplete / done / uncertain
→ Asks: Continue, Close, or Redefine?
3

Run /stop when complete

Atlas enters Settlement Mode to close your project properly.

→ Harvests learnings (decisions, patterns)
→ Runs doubt review (verifies completion)
→ Creates PR and cleans up
→ Archives soul purpose for future reference

Installation

One command. That's it.

Recommended

Skill Mode (Default)

curl -fsSL https://raw.githubusercontent.com/anombyte93/atlas-session-lifecycle/main/install.sh | bash

Installs to ~/.claude/skills/start/. Run /start to begin.

Plugin Mode

curl -fsSL https://raw.githubusercontent.com/anombyte93/atlas-session-lifecycle/main/install.sh | bash -s -- --plugin

Installs to ~/.claude/plugins/atlas-session-lifecycle/ with full plugin structure.

Requirements

  • Claude Code CLI
  • Python 3.8+
  • Git (optional — enables git mv for file organization)
